## 一、角色說明

系統支援**四種角色**，一個帳號可同時擁有多種角色，登入後介面自動展示所有已授權的功能模組。

| 角色 | 名稱 | 主要能做什麼 |
|------|------|------------|
| `admin` | 系統管理員 | 管理所有功能，含組織架構、帳號、場景、評核、系統設定、權限管理 |
| `manager` | 任務管理 | 管理自己被指派部門的場景（新增、編輯、匯入 Excel、更新進度、填寫成效）|
| `evaluator` | 評核管理 | 對指定部門的教官和直屬主管逐指標評分，查看本週待評清單 |
| `chief` | 主管 | 依本部/部門/課別層級查看與管理所屬範圍內的場景，跨本部完全隔離 |

> **多重角色範例**：同一帳號可同時是「課別主管（chief）+ 任務管理（manager）+ 評核管理（evaluator）」，登入後同時看到三個功能模組。
> **executive 唯讀強制**：公司管理層的所有寫入請求（POST/PUT/PATCH/DELETE）在 API 層直接拒絕，不依賴介面按鈕是否顯示。

## 二、登入與認證

| 功能 | 說明 |
|------|------|
| 帳號密碼登入 | 輸入帳號與密碼，驗證通過後取得 JWT Token（8小時有效）|
| 多重角色分流 | 登入後依帳號所擁有的角色，自動顯示對應的導覽選單與功能入口 |
| 登出 | 清除 Token，回到登入頁 |
| 同帳號多角色 | 不需切換，單次登入即可同時使用 manager、evaluator、chief 等所有已授權功能 |
| 首次登入強制修改密碼 | 新建帳號預設密碼為帳號名稱；首次登入會強制展示「修改密碼」對話框，未完成不得進入任何功能頁面 |
| 自行修改密碼 | 所有帳號登入後均可從偏好設定這修改密碼，需輸入舊密碼驗證後才可變更 |

## 三、組織管理

> 適用角色：`admin`  
> 入口：後台管理 → 組織管理

系統採三層組織架構：**本部 → 部門 → 課別**。場景掛在「部門」層，存取控制以部門為單位。

### 3-1 本部（Division）管理

| 功能 | 說明 |
|------|------|
| 新增本部 | 建立頂層組織單位，例如「總經理室」、「勞安室」|
| 編輯本部名稱 | 修改本部顯示名稱 |
| 刪除本部 | 僅當本部下沒有任何部門時才可刪除，防止誤刪 |

### 3-2 部門（Department）管理

| 功能 | 說明 |
|------|------|
| 新增部門 | 在指定本部下建立部門，例如「行政部」|
| 編輯部門名稱 | 修改部門名稱 |
| 刪除部門 | 部門下已有場景時禁止刪除，保護資料完整性 |
| 教官清單維護 | 在部門右側面板新增/編輯/刪除「種子負責人（教官）」名單，供場景填寫時快速選取 |
| 直屬主管清單維護 | 在部門右側面板新增/編輯/刪除「直屬主管」名單，供場景填寫時快速選取 |
| 部門成員名嫊維護 | 在部門右側面板新增/編輯/刪除**部門成員**記錄（姓名、工號選填、職稱）；場景的**任務負責人** / **種子負責人**均可跨部門從全公司成員名嫊選取 |

### 3-3 課別（Section）管理

| 功能 | 說明 |
|------|------|
| 新增課別 | 在指定部門下建立課別，例如「行政課」|
| 編輯課別名稱 | 修改課別名稱 |
| 刪除課別 | 刪除指定課別 |

### 3-4 樹狀圖展示

| 功能 | 說明 |
|------|------|
| 三層樹狀展示 | 頁面以樹狀圖同時呈現本部→部門→課別架構，一覽整體組織 |
| 節點搜尋 | 輸入關鍵字即時篩選樹狀節點 |

> 入口：後台管理 → 使用者管理

### 4-1 帳號基本管理

| 功能 | 說明 |

| 設定密碼 | Admin 重設廣密碼，重設後自動設 mustChangePassword=true，使用者下次登入必須再次修改 |

### 4-2 角色指派

| 功能 | 說明 |
|------|------|
| 設定多重角色 | 從 admin / manager / evaluator / chief 中勾選一或多個角色，同時賦予 |

| 課別主管指派 | 設定哪位帳號是指定課別的主管 |

### 4-5 帳號總覽

| 功能 | 說明 |
|------|------|

## 五、場景管理

> 適用角色：`admin`（全部門）、`manager`（自己部門）  
> 入口：場景管理

每個 AI Agent 任務場景共有 **34 個欄位**，透過右側抽屜（Drawer）分 4 個分頁填寫管理。

### 5-1 場景列表

| 功能 | 說明 |
|------|------|
| 多條件篩選 | 依本部、部門、課別、狀態、優先序、關鍵字快速篩選場景清單 |
| 查看場景詳情 | 點擊場景名稱展開右側 Drawer，檢視所有欄位資訊 |
| 快速更新進度 | 直接在列表拖動進度滑桿或切換狀態，不需開啟 Drawer |

### 5-2 新增 / 編輯場景（4 Tab Drawer）

**Tab 1：基本資訊**

| 欄位 | 說明 |
|------|------|
| 項目編號 | 自訂編號 |
| 場景名稱 ⭐ | 必填，場景核心識別 |
| 所屬本部 | 自動帶入（唯讀），依所選部門自動對應 |
| 所屬部門 | 下拉選擇（manager 只能選自己被指派的部門）|
| 所屬課別 | 依部門動態載入課別清單，下拉選擇 |
| 維持 / 開發(V) / 作廢 | 標記此場景是維持既有 / 開發新(V) / 確定不開發（作廢）|
| 開發方式 | 1.AI Agent / 2.系統開發 / 3.自行開發 / 4.其他工具 |
| 開發工具說明 | 條件顯示：開發方式選「其他工具」時才顯示，填寫工具名稱 |
| AI Agent 用途分類 | 業務分類標籤 |

**Tab 2：需求與執行**

| 欄位 | 說明 |
|------|------|
| (輸入) 常見問項 | 希望 AI 處理什麼問題 |
| (輸出) 預期成果 | 期望 AI 產出的結果形式 |
| 任務步驟或邏輯 | 執行流程說明 |
| 原始資料範例 | 輸入資料的範例說明 |
| 最終資料範例 | 輸出資料的範例說明 |
| 每次執行耗費時間 | 單次執行時間 |
| 每月執行頻率 | 一個月執行幾次 |
| 有需求人數 | 有此需求的人員數 |

**Tab 3：負責人**

| 欄位 | 填寫方式 | 說明 |
|------|---------|------|
| 任務負責人 | 多選下拉 | **可跨部門**從全公司成員名嫊選取；選項顯示「姓名（所在部門）」以辨識同名人員；選定後帶入姓名快照，不因往後更名而失效 |
| 種子負責人 | 多選下拉 | **可跨部門**從全公司成員名嫊選取；顯示方式同上；選定後帶入姓名快照 |
| 直屬主管 | 單選下拉 | 從所屬部門直屬主管清單選取**一位**，代表此任務的課別/部門/本部層級主管 |

**Tab 4：管理與成效**

| 欄位 | 說明 |
|------|------|
| 優先序 | 高 / 中 / 低 |
| 狀態 | 規劃中 / 進行中 / 已完成 / 暫停 |
| 進度 | 0–100% 滑桿 |
| 成立日 | 場景正式成立的日期 |
| 預計完成日 | 目標完成日期 |
| 實際完成日 | 狀態改為「已完成」時自動填入，也可手動修改 |
| 上線日期時間 | 實際上線時間，上線後填寫 |
| 預估節省時數（每月）| 【本部門】導入 AI 後預估每月節省工時（Float）|
| 實際節省時數（每月）| 【本部門】上線後實際量測節省工時（Float）|
| 效率提升% | 【本部門】**系統自動計算**：實際節省時數 ÷ 預估節省時數 × 100%，**唯讀欄位**，不需也不可手動修改；任一數値為空時顯示「—」|
| 實際需求人數 | 上線後實際使用此場景的人數（Int，對比預估値）|
| 文字成效說明 | 質性描述成果 |
| 上線實際成效說明 | 上線後填寫實際成效描述 |
| 其他量化成效 | 其他計量指標 |
| 備註 | 自由備忘 |

**跨部門效益貢獻（可新增多筆）**

> 若此場景對**其他部門**也有效益，可在此處新增受益部門記錄。每個受益部門各填一筆，每筆效益數値獨立計算。

| 子欄位 | 說明 |
|--------|------|
| 受益部門 ⭐ | 下拉選擇受益部門（不可與場景所屬部門重複）|
| 預估月節省時數 | 對該部門預估每月節省工時（Float，選填）|
| 實際月節省時數 | 上線後實際量測（Float，選填）|
| 效率提升% | **系統自動計算**，唯讀 |
| 預估需求人數 | 該部門有此需求的人數（Int，選填）|
| 實際需求人數 | 上線後該部門實際使用人數（Int，選填）|
| 效益說明 | 對該部門的質性效益描述（選填）|
| 備註 | 補充說明（選填）|

### 5-3 刪除

| 功能 | 說明 |
|------|------|
| 軟刪除 | 場景標記為「不啟用」，資料保留不顯示於清單，可復原 |
| 硬刪除（Admin）| 管理員可永久刪除場景紀錄 |

### 5-4 自動機制

| 機制 | 說明 |
|------|------|
| 自動填入完成日 | 狀態切換為「已完成」時，系統自動記錄當天日期為 completedDate |
| 本部唯讀帶入 | 場景 Tab 1 的「所屬本部」欄位依所選部門自動填入，不可手動修改 || 效率提升%自動計算 | 填寫本部門實際節省時數後，自動計算本部門效率提升%，不需手動 |
| 跨部門效益效率自動計算 | 跨部門效益子清單中填入實際月節省時數後，同樣自動計算該受益部門效率提升%（唯讀）|

### 5-5 其他快速更新方式

除了 Drawer 完整表單外，系統提供多種便捷的資料更新途徑：

| 方式 | 適合情境 | 說明 |
|------|---------|------|
| **場景 Drawer（4 Tab）** | 完整新增/修改 | 開啟右側抽屜，填寫全部 34 個欄位 |
| **行內快速編輯** | 快速改単一欄位 | 在場景列表直接點選儲存格即時修改：**進度、狀態、優先序**， Enter 或點擊外部確認 |
| **進度滑桿** | 快速更新百分比 | 場景列表直接拖動進度條，自動儲存 |
| **批次操作** | 多筆同步變更 | 勾選多筆場景 → 批次設定狀態 / 優先序 |
| **Excel 更新匙入** | 大量欄位更新 | 匙出当前場景清單 → 在 Excel 中修改 → 重新匙入，系統自動比對並更新 |

## 六、Excel 匯入

> 適用角色：`admin`（任意部門）、`manager`（自己部門）  
> 入口：場景管理 → 匯入  
> **每次匯入自動記錄操作人員帳號、姓名及匯入時間**

### 6-1 新增匯入

| 功能 | 說明 |
|------|------|
| 選擇目標部門 | 選擇要匯入到哪個部門（manager 只能選自己的部門）|
| 上傳 xlsx 檔案 | 支援標準範本格式，自動對應 18 個欄位到場景資料；項目編號由系統自動生成 |
| 課別自動比對 | Excel 課別欄位依名稱比對所選部門的課別清單，名稱完全相符才自動關聯，找不到則留空 |
| 防重複偵測 | 以「部門+場景名稱」計算雜湊值，已存在的場景自動跳過不重複匯入 |
| 匯入結果報告 | 匯入完成後顯示：成功筆數 / 跳過筆數（防重複）/ 錯誤清單 |
| 匯入記錄 | 每筆匯入操作記錄帳號、姓名、時間、部門、成功/跳過/錯誤筆數，供稽核追蹤 |
| 下載標準範本 | 下載系統提供的 Excel 範本，包含欄位名稱、格式說明與長度限制一覽 |

### 6-2 更新匯入（Excel 更新既有場景）

以**場景名稱**為比對鍵，可透過 Excel 批次更新已存在的場景資料：

| 規則 | 說明 |
|------|------|
| 找到同名場景 | 更新所有「可更新欄位」，保護「不可更改欄位」不被覆蓋 |
| 找不到同名場景 | 自動新增為新場景（同新增匯入邏輯）|
| 更新結果報告 | 顯示：新增筆數 / 更新筆數 / 跳過筆數（驗證未通過）/ 錯誤清單 |

**不可更改欄位（識別鍵）：**
- `sceneName`（場景名稱）— 作為比對鍵本身，不可被覆蓋
- `departmentId`（所屬部門）— 由匯入時選擇的目標部門決定

**可更新欄位：** 以上兩項之外所有 18 欄，涵蓋課別、開發方式、需求描述、負責人等

### 6-3 Excel 欄位對應與限制（18 欄）

> 項目編號（itemNo）由系統自動遞增生成，**不包含在 Excel 欄位中**。  
> Excel 範本第一列顯示完整欄位說明、格式要求與長度限制。

| Excel 欄位 | 對應系統欄位 | 類型 | 最大長度 | 必填 | 說明 |
|-----------|------------|------|---------|------|------|
| 任務場景名稱 ⭐ | sceneName | 文字 | 100 字 | ✅ | 新增/更新比對鍵，不可更改 |
| 課別 | sectionId | 文字 | 依清單 | 選填 | 名稱比對課別清單 |
| 維持/開發(V)/作廢 | maintainOrDevelop | 選項 | 三選一 | 選填 | |
| 開發方式 | developMethod | 選項 | 1/2/3/4 | 選填 | |
| 開發工具說明 | developToolDesc | 文字 | 200 字 | 選填 | 開發方式=其他工具時填 |
| 預估節省時數(每月) | timeSavedHours | 數字 | 9999.9 | 選填 | |
| 每次執行耗費時間 | timePerExecution | 文字 | 50 字 | 選填 | |
| 每月執行頻率 | monthlyFrequency | 文字 | 50 字 | 選填 | |
| 有需求的人數 | demandCount | 整數 | 正整數 | 選填 | 預估需求人數 |
| (輸入)常見問項 | inputDesc | 文字 | 1000 字 | 選填 | |
| (輸出)預期輸出成果 | outputDesc | 文字 | 1000 字 | 選填 | |
| 任務步驟或處理邏輯 | taskSteps | 文字 | 2000 字 | 選填 | |
| 原始資料範例 | rawDataExample | 文字 | 1000 字 | 選填 | |
| 最終資料範例 | finalDataExample | 文字 | 1000 字 | 選填 | |

## 十一、權限管理

> 適用角色：`admin`  
> 入口：後台管理 → 權限管理

核心機制：系統維護一張「**角色 × 功能**授權矩陣表（RoleFeature）。未在矩陣中開啟的功能 = 使用者看不到入口 + API 一律回傳 403。

### 11-1 功能授權矩陣

| 功能鍵 | 功能名稱 | manager | evaluator | chief | executive |
|---------|---------|---------|-----------|-------|-----------|
| `evaluation_view` | 查看評核結果 | 預設關 | 固定開 | 預設關 | 預設關 |
| `evaluation_submit` | 提交評分 | 固定關 | 固定開 | 固定關 | 固定關 |
| `dashboard_view` | 儀表板 | 固定開 | 固定開 | 預設關 | 固定開 |
| `scenes_export` | 場景匹出 | 預設關 | 預設關 | 預設關 | 固定開 |
| `org_view` | 組織結構查看 | 預設關 | 預設關 | 預設關 | 固定開 |
| `users_view` | 使用者與角色查看 | 預設關 | 預設關 | 預設關 | 固定開 |

> **「固定開」** = admin 永遠啟用，不可關閉  |  **「固定關」** = 該角色永遠不得使用  |  **「預設關」** = 預設關閉，可由 Admin 開啟

### 11-2 授權操作

## 十四、推動效率評分系統

> 適用角色：`admin`（評分項目管理+全笹查閱）、`evaluator`（實際評分作業）、`executive`（查看結果）  
> 入口：評分系統  
> 對象：組織層級（公司/本部/部門/課別）+ 人員層級（任務負責人/種子負責人/直屬主管）

> **評分与現有「評核模組」的區別**：評核模組評的是種子負責人/直屬主管的「能力表現」；
> 推動效率評分評的是各層組織與人員對 AI 這件事的「推動动能與成效表現」，兩者相輔相成。

### 14-1 評估週期管理

| 功能 | 說明 |
|------|------|
| 新增週期 | 建立評估週期，設定名稱、開始日、截止日、軚止日（預設当週周三）|
| 開放/關閉評估 | 開放後接受評分從業人員提交；關閉後禁止新的確認提交 |
| 週評 / 月評切換 | 需評頻率可局設：週一次或月一次，選擇待評範圍模式 |
| 編輯/刪除週期 | 調整週期設定 |

### 14-2 評分項目管理

| 功能 | 說明 |
|------|------|
| 新增評分項目 | 建立評分指標，設定名稱/說明/満分/適用範圍/加權 |
| 區分系統可自動計分 | 標記為「系統計分」的項目，附註計算公式說明（供評分員了解依據） |
| 需人工評分項目 | 无法量化的項目（如：推動積極度、跨部門協作）由評分員直接輸分 |
| 操作項目 | 編輯/停用/啟用評分項目 |

### 14-3 系統自動預填分數

| 功能 | 說明 |
|------|------|
| 自動計算量化項目 | 系統依場景資料自動算出：場景完成率、完工場景完成率、實際效率提升%平均、負責場景進度平均 |
| 計算依據明細 | 評分員查看每項系統分的計算方式，如「完工 14/20 場 = 70%，得 7 分」 |
| 為尚未確認的評估自動产生長本 draft | 開放期間系統將全部的被評對象就自動計算系統預填分，資料為 pending 狀態 |
| 手動觸發重算 | 有需要時可手動重新觸發系統計分 |

### 14-4 評分作業（評分員使用）

| 功能 | 說明 |
|------|------|
| 自選評估對象 | 評分員自行决定要評哪個組織或人員，不強制分配 |
| 查看系統預填分 | 打開任一評估對象，即可看到系統預算的各項分數與計算依據 |
| 逐項確認/覆寫 | 對系統可計分項目：直接接受系統分，或輸入自己的覐寫分數；對需人工項目：直接輸入分數 |
| 整體増減分 | 確認總分是否合理，可填入整體増減分（正負均可）並輸入說明 |
| 提交確認 | 確認後 status 改為 confirmed，建立正式記錄 |
| 重新評分 | 週期開放期間可重新評分（覆寫先前評估）|

### 14-5 評分結果與成長統計

| 功能 | 說明 |
|------|------|
| 評分結果統計 | 展示指定週期內各組織/人員總分排名，可鐵取比較分碹明細 |
| 成長趨勢視覺化 | 選定組織或人員，查看其跨期總分趨勢圖，觀察成長軌跡 |
| 比較查看 | 同一期內多個組織或人員的評分橫向比較 |
| 評分人員進度 | 管理層可查看本期哪些組織/人員已評分、哪些尚未評，提醒評分員 |
| Executive 查看 | Executive 可查看全公司推動效率評分結果與成長統計（唯讀）|

| 機制 | 說明 |
|------|------|
| 自動完成日 | 場景狀態切換為「已完成」時，系統自動寫入當天日期為實際完成日 |
| 防重複匯入 | Excel 匯入時以「部門 + 場景名稱」計算唯一雜湊值，相同資料自動跳過並回報 |
| 軟刪除保留 | 場景刪除預設為軟刪除（標記 active=false），資料不消失，管理員可硬刪除 |
| 部門 / 本部隔離 | manager 只能操作自己被指派的部門；chief 依管轄節點自動計算可存取範圍，跨本部完全阻擋 |
| 密碼安全 | 密碼以 bcrypt（salt: 12）雜湊儲存，系統不保存明文 |
| JWT 安全 | Token 8 小時後自動失效，需重新登入 |
| 本部唯讀帶入 | 場景的所屬本部欄位依部門自動帶入，不可手動修改，避免資料錯誤 |
